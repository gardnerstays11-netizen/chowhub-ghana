1. Project Overview

ChowHub Ghana is a full-stack food and dining discovery platform built for the Ghanaian market. It enables users to discover restaurants, bars, cafes, and street food vendors across Ghana, place orders (via WhatsApp or online payment), make reservations, and leave reviews.

Key Features

  • Location-based discovery — Find restaurants, bars, lounges near you with GPS-based "Near Me" functionality
  • Advanced food search — Search by dish name across all restaurant menus, with category and occasion filters
  • Dual ordering — Place orders via WhatsApp or pay online via Paystack (Mobile Money, cards)
  • Table reservations — Book tables at restaurants that accept reservations
  • Vendor portal — Restaurant owners manage menus, events, photos, analytics, and respond to orders
  • Admin portal — Full site management: categories, subscriptions, editor's picks, site settings, SEO
  • Mobile app — Native Expo app with onboarding, push notifications, and full feature parity
  • SEO optimized — Dynamic sitemap, JSON-LD, OpenGraph, per-page meta tags

User Roles

VendorRestaurant/bar ownerManage listing, menu, events, photos, view analytics, handle orders/reservations

7. Authentication & Authorization

JWT Implementation

  • Tokens are generated using jsonwebtoken with the SESSION_SECRET env var
  • Passwords hashed with bcrypt (10 salt rounds)
  • Three separate auth flows: user, vendor, admin
  • Auth middleware extracts JWT from Authorization: Bearer <token> header
  • Mobile stores JWT in AsyncStorage; web stores in memory (per-session)

Role-Based Access

MiddlewareUsed For

8. Payment Integration (Paystack)

Overview

Paystack handles all online payments. It supports:

  • Mobile Money — MTN MoMo, Vodafone Cash, AirtelTigo Money
  • Card payments — Visa, Mastercard
  • Bank transfers — Direct bank payment

Payment Flow

  1. User selects "Pay Online" at checkout and places order
  2. Frontend calls POST /api/payments/initialize with order details
  3. Backend creates payment record and calls Paystack API to initialize transaction
  4. User is redirected to Paystack checkout page (card/MoMo/bank)
  5. On completion, Paystack redirects to /payment/verify?reference=xxx
  6. Frontend calls GET /api/payments/verify/:reference to confirm
  7. Paystack sends webhook to POST /api/payments/webhook (HMAC-SHA512 verified)
  8. Backend updates order/subscription payment status

Subscription Payments

  1. Vendor selects a subscription plan
  2. Frontend calls POST /api/payments/initialize-subscription
  3. Same Paystack flow as above
  4. On success, vendor's plan and expiry date are updated
Important: Set PAYSTACK_SECRET_KEY and PAYSTACK_PUBLIC_KEY environment variables. Test keys start with sk_test_ / pk_test_, live keys with sk_live_ / pk_live_.

13. SEO Strategy

  • Dynamic Sitemap: GET /api/sitemap.xml generates XML sitemap from all active listings
  • robots.txt: Allows all crawlers, references sitemap
  • Per-page Meta Tags: usePageMeta hook sets title, description, OG, Twitter Card, canonical URL
  • JSON-LD: Organization schema on homepage, Restaurant schema on listing detail
  • OpenGraph: Full OG metadata for social sharing
  • Admin-configurable: SEO settings manageable from admin panel (meta tags, OG image, keywords)

17. Known Issues & Future Work

Known Issues

  • POST /storage/uploads/request-url has no auth middleware — security concern for file uploads
  • Web auth stores user data in memory only (no localStorage persistence) — refreshing clears login
  • Wouter v3 useLocation() strips query strings — must use window.location.search for URL params

Future Enhancements

  • Real-time order tracking with WebSocket/SSE
  • Push notification delivery for order status updates
  • Vendor payout system via Paystack Transfers
  • Multi-language support (English, Twi, Ga, Ewe)
  • Loyalty/rewards program
  • Social features (food photos, follow restaurants)
  • Advanced analytics dashboard with revenue reports
  • Automated vendor plan expiry and renewal reminders
