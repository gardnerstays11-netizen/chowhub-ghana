Build a full-stack food and dining discovery platform for the Ghanaian 
market called [APP_NAME]. It is a platform where users can find every 
type of food and dining establishment across Ghana — restaurants, fine 
dining, breakfast spots, brunch spots, chop bars, bars, lounges, cafes, 
bakeries, food stalls, fast food joints, hotel dining, buffets, 
rooftop dining, pop-ups, and more.

Users can discover places near them, view full listings, make 
reservations, place orders, and contact businesses directly via 
phone, website, WhatsApp, or social media.

There is no payment processing on the platform for now. All contact 
and transactions happen directly between the user and the business.

Choose the best modern tech stack for this project. Pick technologies 
that are:
- Production ready and widely supported
- Best suited for a discovery/listing platform with maps, 
  filters, and SEO
- Scalable as the platform grows
- Easy to deploy and maintain
- Good for both web and mobile

Document your tech stack choices at the top of the project README 
and explain why you chose each technology.

## DATABASE SCHEMA

Create the following tables:

### users
- id (uuid, primary key)
- name (string)
- email (string, unique)
- phone (string)
- password (hashed)
- city (string)
- created_at (timestamp)

### vendors
- id (uuid, primary key)
- business_name (string)
- email (string, unique)
- phone (string)
- password (hashed)
- status (enum: pending, approved, rejected)
- plan (enum: free, premium)
- plan_expires_at (timestamp, nullable)
- created_at (timestamp)

### listings
- id (uuid, primary key)
- vendor_id (foreign key → vendors)
- name (string)
- slug (string, unique) — e.g. "kfc-accra-mall"
- description (text)
- category (enum: 
    restaurant,
    fine_dining,
    breakfast_brunch,
    chop_bar,
    bar_lounge,
    rooftop_bar,
    cafe_bakery,
    fast_food,
    food_stall,
    buffet,
    hotel_dining,
    dessert_drinks,
    pop_up,
    food_court
  )
- cuisine_type (array of strings) — e.g. ["Ghanaian", "Chinese", 
  "Continental", "Lebanese", "Indian", "Italian", "American", 
  "Japanese", "West African", "Fusion"]
- dining_style (enum: casual, formal, fine_dining, buffet, 
  self_service, takeaway, delivery, dine_in_only)
- meal_period (array of enums: breakfast, brunch, lunch, 
  dinner, all_day, late_night)
- price_range (enum: budget, moderate, upscale, luxury)
  — represented as: GHS 0-50, GHS 50-150, GHS 150-300, GHS 300+
- city (enum: accra, kumasi, takoradi, tamale)
- area (string) — e.g. "Osu", "East Legon", "Adum"
- neighbourhood (string)
- address (string)
- landmark (string) — e.g. "opposite Accra Mall"
- lat (float)
- lng (float)
- phone (string)
- whatsapp (string)
- website (string, nullable)
- instagram (string, nullable)
- twitter (string, nullable)
- facebook (string, nullable)
- tiktok (string, nullable)
- opening_hours (JSON) — per day open/close times
- features (array of strings) — e.g. ["Live Music", 
  "Outdoor Seating", "Private Dining", "Parking", "AC", 
  "WiFi", "Rooftop", "Sports Viewing", "Kids Friendly", 
  "Wheelchair Accessible", "Reservations Required", 
  "Takeaway Available", "Delivery Available", "Shisha", 
  "Pool Side", "Sea View", "Garden Seating", "BYOB", 
  "Halal", "Vegan Options"]
- dress_code (enum: casual, smart_casual, formal, none)
- accepts_reservations (boolean)
- accepts_orders (boolean)
- average_rating (float)
- total_reviews (integer)
- is_featured (boolean, default false)
- is_verified (boolean, default false)
- meta_title (string)
- meta_description (string)
- status (enum: active, inactive, pending)
- created_at (timestamp)

### listing_photos
- id (uuid)
- listing_id (foreign key → listings)
- url (string)
- is_cover (boolean)
- display_order (integer)

### menu_items
- id (uuid)
- listing_id (foreign key → listings)
- name (string)
- description (string)
- price (float, nullable)
- category (string) — e.g. "Starters", "Mains", "Drinks", 
  "Desserts", "Breakfast", "Sides"
- photo_url (string, nullable)
- is_available (boolean, default true)
- is_popular (boolean, default false)

### reservations
- id (uuid)
- listing_id (foreign key → listings)
- user_id (foreign key → users)
- date (date)
- time (string)
- party_size (integer)
- occasion (string, nullable) — e.g. "Birthday", "Anniversary",
  "Business Lunch", "Date Night"
- special_requests (text, nullable)
- status (enum: pending, confirmed, declined)
- created_at (timestamp)

### orders
- id (uuid)
- listing_id (foreign key → listings)
- user_id (foreign key → users)
- items (JSON) — array of menu items and quantities
- order_type (enum: delivery, pickup, dine_in)
- delivery_address (string, nullable)
- note (text, nullable)
- status (enum: pending, confirmed, rejected, completed)
- created_at (timestamp)

### reviews
- id (uuid)
- listing_id (foreign key → listings)
- user_id (foreign key → users)
- rating (integer, 1-5)
- food_rating (integer, 1-5)
- service_rating (integer, 1-5)
- ambience_rating (integer, 1-5)
- value_rating (integer, 1-5)
- comment (text)
- visited_for (string) — e.g. "Dinner", "Birthday", "Business"
- created_at (timestamp)

### saved_places
- id (uuid)
- user_id (foreign key → users)
- listing_id (foreign key → listings)
- created_at (timestamp)

### subscriptions
- id (uuid)
- vendor_id (foreign key → vendors)
- plan (enum: free, premium)
- start_date (timestamp)
- end_date (timestamp)
- status (enum: active, expired, cancelled)

## KEY BEHAVIOURS

Location/Near Me:
- Request GPS on homepage
- Show listings within radius sorted by distance
- Manual city selection as fallback

Slug generation:
- Auto-generate from name + city
- Ensure uniqueness

Opening Hours:
- JSON per day
- Open/Closed badge based on Ghana time (GMT+0)
- Show "Opens at X" or "Closes at X"

Reservation flow:
1. User fills form — status: pending
2. Vendor confirms or declines in inbox
3. User sees status update in dashboard

Order flow:
1. User selects items and order type
2. Adds delivery address if delivery
3. Vendor confirms, rejects, or completes
4. User sees status update

Reviews:
- Logged in users only
- One per user per listing
- Overall + sub-ratings (food, service, ambience, value)
- Average updates automatically

Search:
- All filters combinable
- URL updates with filter params (shareable links)
- Active filters shown as removable chips
- Result count always visible
